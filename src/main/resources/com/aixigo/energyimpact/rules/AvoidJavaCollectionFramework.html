In recent research (see Improving energy-efficiency by recommending Java collections) it has been discovered that ArrayList and other collections are less energy-efficient than the implementations of Eclipse Collections or Apache Commons. In summary, using ArrayList may lead to higher Carbon footprint of your software. “Among all 724 recommendations made to desktop systems, 666 used alternative implementations to JCF, 294 from Apache Commons and 372 from Eclipse Collections. Once again it is possible to observe a trend of replacing well-known collections such as ArrayList, Hashtable, and HashMap by more energy-efficient but less-known alternatives.”

Noncompliant Code Example

An ArrayList is less energy-efficient: 
   List listItems = new ArrayList<>();
A HashMap is less energy-efficient: 
   Map mapItems = new HashMap<>();

Compliant Solution

A FastList is more energy-efficient than ArrayList: 
   List listItems = new org.eclipse.collections.impl.list.mutable.FastList();
A HashedMap is more energy-efficient than HashMap: 
   Map mapItems = org.apache.commons.collections4.map.HashedMap<>();
